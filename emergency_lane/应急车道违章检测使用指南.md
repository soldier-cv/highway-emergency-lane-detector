# 高速公路应急车道违章检测系统 - 使用指南

## 场景说明

**你的场景**：三车道高速公路，拍摄车在第三车道（最右侧行车道），右侧为应急车道。

```
|  车道1  |  车道2  |  车道3(拍摄车)  |  应急车道  |
|  ←左    |  ←中    |  ←右(你在这里)  |  ←目标区域 |
```

系统会检测进入应急车道区域的车辆，识别其车牌号，并记录违章信息。

---

## 快速使用

### GUI 界面（推荐）

```bash
python emergency_lane/traffic_violation_gui.py
```

在界面中选择视频文件，调整参数后点击"开始检测"。

### 命令行运行

```bash
# 主程序（15s剪辑 + 车牌颜色 + 时间戳）
python emergency_lane/run_gpu_v8.py "视频文件路径.mp4"
```

### 调整应急车道区域

应急车道区域默认占画面右侧 16%（从 84% 位置到 100%），如果视频中应急车道位置不同，需要修改脚本中的 `LANE_START_X` 常量。

**参数图解**：
```
┌──────────────────────────────────┐
│              lane_top=0.15       │  ← 忽略上方15%（远处目标小）
│  ┌────────────────────┬─────────┐│
│  │                    │ 应急车道 ││
│  │    正常行驶区域     │ LANE_   ││
│  │                    │ START_X ││
│  │                    │ =0.84   ││
│  │                    │         ││
│  └────────────────────┴─────────┘│
└──────────────────────────────────┘
```

## 输出文件

处理完成后，会生成以下文件（保存在视频同目录的 `{视频名}_检测结果/` 下）：

| 文件 | 说明 |
|------|------|
| `举报视频片段/` | 15秒 FFmpeg 无损剪辑片段 |
| `违章截图/` | 4K 原图截图 |
| `违章检测报告.html` | 含内嵌截图的 HTML 报告 |
| `违章检测报告.json` | 结构化 JSON 数据 |

---

## 技术架构

```
视频帧 → YOLOv12车辆检测(OpenVINO GPU) → 应急车道区域判定 → HyperLPR3车牌识别(OpenVINO GPU)
                                                         ↓
                                              HSV颜色识别 → 15s剪辑 → HTML/JSON报告
```

| 模型 | 用途 | GPU速度 |
|------|------|---------|
| YOLOv12s | 车辆检测 | ~5ms/帧 |
| HyperLPR3 | 车牌识别 | ~27ms/次 |

---

## 调优建议

1. **应急车道区域不准**：修改 `LANE_START_X` 值，用 GUI 预览效果
2. **车牌识别不到**：降低 `CONF_THRESH` 到 0.2，或提高检测缩放 `DET_SCALE`
3. **误检太多**：提高 `MIN_OVERLAP` 到 0.5（只有车身大部分在应急车道才报警）
4. **同一车重复记录**：增大 `COOLDOWN_FRAMES` 值
